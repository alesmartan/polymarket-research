# SEO & ASO Strategy

## Executive Summary

This document outlines a comprehensive Search Engine Optimization (SEO) and App Store Optimization (ASO) strategy for a prediction market platform. With over 58% of new crypto users joining via educational content, and app discovery heavily influenced by store rankings, these organic channels represent critical growth levers.

The crypto SEO landscape is uniquely challenging due to advertising restrictions and regulatory concerns, making organic search a primary acquisition channel. Well-executed SEO can achieve 280%+ increases in organic conversions, while ASO can drive thousands of installs monthly without paid advertising.

# Part 1: SEO Strategy

## Keyword Research Methodology

### Research Process

#### Step 1: Seed Keyword Generation

**Sources for Seed Keywords:**
- Competitor websites (Polymarket, Kalshi)
- Industry publications
- Reddit/forum discussions
- Customer support queries
- Google autocomplete suggestions
- Related searches
- Team brainstorming

**Initial Seed Keywords:**
```
Prediction markets
Prediction market platform
Bet on elections
Crypto betting
Sports prediction markets
Political betting
Event contracts
Decentralized prediction markets
Bet on sports online
Forecast markets
```

#### Step 2: Keyword Expansion

**Tools:**
| Tool | Purpose | Priority |
|------|---------|----------|
| Ahrefs | Volume, difficulty, SERP analysis | Primary |
| SEMrush | Competitor keywords, gaps | Primary |
| Google Keyword Planner | Volume estimates | Secondary |
| AnswerThePublic | Question keywords | Secondary |
| AlsoAsked | Related questions | Secondary |
| Google Trends | Trending topics | Monitoring |

**Expansion Methods:**

1. **Competitor Keyword Gap Analysis**
   - Export competitor rankings
   - Identify keywords you don't rank for
   - Prioritize by volume and relevance

2. **Related Keywords**
   - Google "Searches related to"
   - "People also ask" questions
   - Topic clusters

3. **Long-Tail Expansion**
   - Add modifiers: "how to", "best", "vs", "[year]"
   - Add locations: "USA", "legal"
   - Add intent: "guide", "tutorial", "review"

4. **Question Keywords**
   - "What is prediction market"
   - "How do prediction markets work"
   - "Why are prediction markets accurate"

#### Step 3: Keyword Qualification

**Qualification Criteria:**

| Criterion | Minimum Threshold |
|-----------|-------------------|
| Monthly search volume | 50+ (initially), 100+ (scale) |
| Keyword difficulty | <70 (initially), <80 (scale) |
| Relevance to platform | High/Medium |
| Commercial intent | Any (balanced portfolio) |

**Intent Classification:**
- **Informational:** Learning, researching (how, what, why)
- **Commercial:** Comparing, evaluating (best, vs, review)
- **Transactional:** Ready to act (buy, sign up, platform)
- **Navigational:** Looking for specific site (brand + keyword)

#### Step 4: Prioritization Matrix

**Scoring Model:**

| Factor | Weight | Score (1-5) |
|--------|--------|-------------|
| Search volume | 25% | |
| Keyword difficulty | 25% | |
| Business relevance | 20% | |
| Conversion potential | 20% | |
| Competition | 10% | |

**Priority Tiers:**
- Tier 1 (Score >4): Target immediately, dedicated pages
- Tier 2 (Score 3-4): Target in first 90 days
- Tier 3 (Score 2-3): Target in 6 months
- Tier 4 (Score <2): Monitor, opportunistic

## On-Page Optimization

### Page Structure Best Practices

#### Title Tag Optimization

**Format:** Primary Keyword | Secondary Keyword | Brand

**Character Limit:** 50-60 characters

**Examples:**
```
Homepage:
"Prediction Markets | Trade on Real-World Events | [Brand]"

Landing Page:
"Sports Prediction Markets - Trade Sports Outcomes | [Brand]"

Blog Post:
"How Do Prediction Markets Work? Complete Guide (2025)"

Comparison:
"[Brand] vs Polymarket: Which Platform is Better? (2025)"
```

**Best Practices:**
- Primary keyword first
- Include brand name
- Use numbers when relevant (2025, "5 Ways")
- Create urgency/value ("Complete", "Ultimate")
- Avoid keyword stuffing

#### Meta Description Optimization

**Character Limit:** 150-160 characters

**Format:** Value proposition + CTA + Differentiator

**Examples:**
```
Homepage:
"Trade on real-world events with [Brand] prediction markets.
Sports, politics, crypto & more. Start trading with $0 fees.
Join 100,000+ traders today."

Blog Post:
"Learn how prediction markets work and why they're more
accurate than polls. Complete beginner's guide with examples,
strategies, and tips. Start trading today."
```

#### Header Structure (H1-H6)

**Best Practices:**
```html
<h1>Primary Keyword - Main Page Title</h1>
(Only one H1 per page)

<h2>Secondary Keyword - Major Section</h2>
(Multiple H2s for main sections)

<h3>Supporting Keyword - Subsection</h3>
(Multiple H3s within H2 sections)

<h4>Detailed Point - If needed</h4>
```

**Example Structure:**
```
H1: Complete Guide to Sports Prediction Markets

H2: What Are Sports Prediction Markets?
  H3: How Sports Prediction Markets Differ from Sportsbooks
  H3: Types of Sports Markets Available

H2: How to Trade Sports Predictions
  H3: Creating Your First Trade
  H3: Understanding Odds and Probabilities

H2: Best Sports Prediction Strategies
  H3: Value Betting Approach
  H3: Momentum Trading
```

### Content Optimization Checklist

**For Every Page:**

- [ ] Primary keyword in H1
- [ ] Primary keyword in first 100 words
- [ ] Primary keyword in at least one H2
- [ ] 2-3 secondary keywords naturally included
- [ ] Keyword density 1-2% (not forced)
- [ ] Synonyms and related terms used
- [ ] 300+ words minimum (2000+ for pillar content)
- [ ] Unique, valuable content
- [ ] Clear call-to-action

### URL Structure

**Best Practices:**
- Short and descriptive
- Include primary keyword
- Use hyphens, not underscores
- Lowercase only
- Avoid parameters when possible

**Examples:**
```
Good:
/prediction-markets
/sports-prediction-markets
/how-do-prediction-markets-work
/polymarket-vs-kalshi-comparison

Bad:
/page123
/sports_prediction_markets
/blog/2025/01/15/post
/index.php?page=markets&type=sports
```

### Image Optimization

**File Names:**
```
Good: prediction-market-trading-interface.png
Bad: IMG_1234.png
```

**Alt Text:**
```
Good: "Prediction market trading interface showing
      Super Bowl odds"
Bad: "image" or "screenshot"
```

**Technical:**
- Compress images (WebP preferred)
- Lazy loading for below-fold images
- Responsive images (srcset)
- Max width: 1200px for full-width
- Target file size: <100KB

### Internal Linking

**Strategy:**
- Every page links to 3+ related pages
- Use descriptive anchor text (not "click here")
- Link from high-authority pages to new pages
- Create topic clusters with pillar/cluster structure

**Topic Cluster Example:**
```
PILLAR PAGE: Complete Guide to Prediction Markets

CLUSTER PAGES:
├── How Prediction Markets Work
├── Prediction Market Strategies
├── Sports Prediction Markets
├── Political Prediction Markets
├── Crypto Prediction Markets
├── Prediction Markets vs Betting
├── Best Prediction Market Platforms
└── Prediction Market Regulations
```

## Technical SEO Checklist

### Site Speed

**Target Metrics:**
| Metric | Target | Tool |
|--------|--------|------|
| Largest Contentful Paint (LCP) | <2.5s | PageSpeed Insights |
| First Input Delay (FID) | <100ms | PageSpeed Insights |
| Cumulative Layout Shift (CLS) | <0.1 | PageSpeed Insights |
| Time to First Byte (TTFB) | <600ms | WebPageTest |

**Optimization Actions:**
- [ ] Enable compression (gzip/brotli)
- [ ] Minify CSS/JS
- [ ] Optimize images (WebP, lazy load)
- [ ] Enable browser caching
- [ ] Use CDN
- [ ] Reduce server response time
- [ ] Defer non-critical JavaScript
- [ ] Preload critical resources

### Mobile Optimization

**Requirements:**
- [ ] Responsive design (all breakpoints)
- [ ] Mobile-first CSS approach
- [ ] Tap targets >48px
- [ ] Readable font size (16px+ base)
- [ ] No horizontal scrolling
- [ ] Fast mobile load time (<3s)
- [ ] Avoid intrusive interstitials

### Indexing & Crawlability

**Sitemap:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://platform.com/</loc>
    <lastmod>2025-01-15</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://platform.com/markets</loc>
    <lastmod>2025-01-15</lastmod>
    <changefreq>hourly</changefreq>
    <priority>0.9</priority>
  </url>
  <!-- Additional URLs -->
</urlset>
```

**Robots.txt:**
```
User-agent: *
Allow: /

Disallow: /admin/
Disallow: /api/
Disallow: /private/
Disallow: /*?*ref=
Disallow: /*?*utm_

Sitemap: https://platform.com/sitemap.xml
```

**Technical Checklist:**
- [ ] XML sitemap submitted to Search Console
- [ ] Robots.txt properly configured
- [ ] No blocked critical resources
- [ ] No orphan pages
- [ ] Proper canonical tags
- [ ] No duplicate content
- [ ] 301 redirects for changed URLs
- [ ] Custom 404 page
- [ ] Hreflang for international (if applicable)

**Target Topics:**

- Sports betting
- Elections/polling

**Monthly Goal:** 2-4 media placements

[Additional insight or example - 2-3 sentences]

Source: [Your Name], [Title] at [Platform Name]
- [One-line bio/credential]

#### 5. Broken Link Building

**Process:**
1. Find broken links on relevant sites
2. Create or identify content that could replace

- Check My Links (Chrome extension)
- Screaming Frog

```

[your URL]

Best,
[Your Name]
```

- ASO provides sustainable, free acquisition
- Critical for crypto apps facing ad restrictions

| Factor | Apple App Store | Google Play Store |
|--------|-----------------|-------------------|
| Title length | 30 characters | 30 characters |
| Subtitle | 30 characters | N/A |
| Short description | N/A | 80 characters |

**Best Practices:**
- Separate keywords with commas
- No spaces after commas
- Single words (plurals covered automatically)
- No brand name (already indexed)
- No category words (already indexed)

**Keyword Research Process:**
1. Brainstorm seed keywords
2. Research in App Store
3. Analyze competitor keywords
4. Use ASO tools (Sensor Tower, AppTweak)
5. Prioritize by volume and relevance

**Example Keyword Set:**
```
prediction,market,trading,bet,odds,sports,politics,
election,crypto,forecast,probability,trade,events,
betting,gamble,predict
```

- Include 2-3x in first paragraph

- Feature 1 with keywords

- Feature 3 with keywords

## Review Management

### Review Acquisition Strategy

**In-App Prompts:**
```
Trigger after positive actions:
- First winning trade
- Deposit completion
- 5th successful trade
- Withdrawal completed

Prompt: "Enjoying [Brand]? Share your experience!"
[Rate Now] [Maybe Later] [No Thanks]
```

**Timing Rules:**
- Don't prompt too early (<3 sessions)
- Don't prompt during negative moments
- Limit to 1 prompt per month
- Respect "No Thanks" responses

### Review Response Strategy

**Response to Positive Reviews:**
```
Thanks for the great review, [Name]! We're glad you're
enjoying [specific feature they mentioned]. Happy trading!
```

**Response to Negative Reviews:**
```
Hi [Name], we're sorry to hear about your experience.
We take [issue] seriously. Please contact support@platform.com
so we can help resolve this. Thank you for the feedback.
```

**Response Guidelines:**
- Respond within 24-48 hours
- Address specific concerns
- Never be defensive
- Move complex issues offline
- Thank all reviewers
- Update responses when issues resolved

## A/B Testing

### Testing Priority

1. **Icon** (highest impact)
2. **Screenshots** (high impact)
3. **Title/Subtitle** (medium impact)
4. **Description** (lower impact)

### A/B Test Framework

**Test Duration:** 2-4 weeks minimum
**Traffic Split:** 50/50
**Statistical Significance:** 95%+

**Google Play Experiments:**
- Native A/B testing available
- Test up to 5 variants
- Automatic winner selection

**iOS Testing:**
- Use App Store Connect Product Page Optimization
- Test up to 3 treatments
- Limited to screenshots, videos, icons

### Test Ideas

**Icon Tests:**
| Variant | Hypothesis |
|---------|------------|
| A: Abstract design | Modern, memorable |
| B: Chart/graph icon | Shows trading |
| C: Text logo | Brand recognition |
| D: Mascot/character | Approachable |

**Screenshot Tests:**
| Test | Variants |
|------|----------|
| First screenshot | Feature vs. social proof vs. CTA |
| Text size | Large vs. medium text |
| Style | Light vs. dark UI |
| Content | Sports focus vs. politics vs. general |

### Measurement

**Primary Metrics:**
- Conversion rate (impressions → downloads)
- Qualified downloads (with positive retention)

**Secondary Metrics:**
- Page views
- Browse vs. search conversion
- Geographic performance
